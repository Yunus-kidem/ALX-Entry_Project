    Database – Store your data
    • Whenever you create a backend for your application you mostly want to manage your data, create an API and create multiple end points for it. so that user can get access to it for this you will need Data base
      • Relational Database – PostgreSQL (easy to learn and highly support Djnago), MySQL
      • NoSQL Database:
        • Key value store: Redis, DynamoDB (AWS service also super-fast)
        • Document base: MongoDB (looks like JSON )
      • Cloud Database: Amazon RDS , Firebase
    • Multi-model Database: Cosmos DB (huge database, kind of connects every database for example in one you may have relational database and key value store.
    Rest API vs GraphQL
    • Rest API - You create a multiple end points and you expose your data with different kind of structures.
    • In GraphQL – you only have one end point, you don’t have to remember all of your endpoints you just have to ask one end point and front end developer can ask for what data he wants.
      • Query language, you describe how to as for data
      • It lets the client specify exactly what data it needs
      • It makes it easier to aggregate data from multiple sources.
      • You can use APOLLO to develop it. also super-easy to create in Django.
      • If you go to Facebook developer you can get the GraphQL API and will continue by your own.

    Web developer stacks

    1. LAMP (Linux, Apache, MySQL, PHP) – The old-school stack
    2. MEAN (Mongo DB, ExpressJS, AngularJS, NodeJS – The jock stack

    4. MERN MongoDB Express React Node.js – most popular

    • Deploy apps on server (AWS (hard but great), Heroku (super-easy; without having really know the details. Good for beginners), Azure, Google cloud)
    • Docker containers
    • Automated testing
    • CI/CD – to make sure your app don’t crush. You will run in a thing like a pipeline and if it’s green your good to go. And also other developers could see your code and will suggest you.
