Front end development Road map

  1. HTML/HTML5

    • Basic & boring texture

  2. CSS

    • Make things more beautiful
      • Basics (syntax, color, lists, tables…)
      • Box model (Margin, Padding, Border)
      • Advanced (Position, Layers, Grid..)

  3. Java Script

    • More fancier things/ interactivity
    • To learn java script: basic path

These are the 3 major things you have to learn to become a front-end developer.



HTML, CSS and JS popular Libraries – highly recommended, it make it so damn easy for app
  • Bootstrap 4
  • Materialize
  • Bulma

Just choose one library that suits you and you’re good to go. It will make your life much easier but make sure to master the top 3 before starting to use libraries.

Learn about responsive layout
  • Setting the view point

Learn about Dev tools

Dev tools allows you to really dig deep into a page and be able to debug it, be able to change it on the fly, add different colors, check the styling, fix a new problem, look what’s going on the network/ performance tab. Console logging. Important tools you have to learn as a developer.

You have to Master the command line; you will lack really fundamental skills if you skip this, you can’t problem solve or debug things accurately. You will learn to love it after getting use to it.

Frontend frameworks! React js 21.5% Vue 16.1% angular js. 12.2%

Vue.js
  • Have a huge community, huge amount of stars in GitHub. Easiest framework to start learning.
  • Component-based and visual DOM; will make it very fast.
    • i. Simple syntax
    • ii. Beginner friendly
    • iii. Rapid grow of community
    • iv. Not for complex projects, it’s a Light app
  • Popular companies using this; gitlab, alibaba,
  • Another cons: it is not maintained by one of the popular companies like Google/facebook
Angular.js
  • Hardest to learn
  • Component based framework, unlike react it is a 2-way data binding.
    • Strong community
    • Simple for single page app
    • Complex and not easy to learn
  • Very good for huge projects like banks
  • Popular companies: Google, XBOX, BMW
React.js
  • Easiest to learn, compared to angular. #1 popular
    • Maintained by Facebook
    • High performance
    • Not SEO friendly
  • Popular companies; NETFLIX, New york Time, Yahoo, Uber, PayPal


State management
  • Use for larger apps when you need to share data across multiple components. It makes it easier to build more maintainable code, and easier to track down the root cause when something goes wrong.
    • React – Redux, React Hooks and Context API
    • Vue – Vuex
    • Angular – NgRx
Static Site generators

Everything that is runned by static website is super-fast. Generated by HTML,CSS, JS. You don’t have to run everything on a server which means it’s also super-secure when ever someone hacks your website there won’t be serious damage for example don’t get your data or credit card. And great for SEO they will be ranked high on Google.

  • NextJS (react JS)
  • Gatsby (react JS) – another cool thing you can get data from anywhere other APIs. It has a great community, super easy to learn just by using documentation and also it’s free
  • Hugo (GO)
  • Nuxt (Vue JS)

First your run the Nodejs and it generates all of the content for GatsbyJS and then everything you surf are using static website.

  • Super fast
  • Great SEO
  • Many plugins
  • Secure
  • Easy scaling
  • Free hosting
